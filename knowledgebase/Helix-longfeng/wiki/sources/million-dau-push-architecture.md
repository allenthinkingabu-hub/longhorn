## Scope & thesis

The article argues that a single, standardized-template push platform can serve 10万+ QPS at ≤200 ms latency across four terminal types, if you:

1. Split the system into four layers (access / core service / channel / data) and four microservices inside the core layer.
2. Template messages with JSON Schema for validation + i18n content blocks.
3. Use vendor-native push channels first, fall back to third parties, and cache on the server only for web.
4. Schedule tasks with a hybrid of PowerJob (real-time / cron) and Elastic-Job (daily broadcast).
5. Enforce observability, reliability, and compliance as first-class architectural concerns.
